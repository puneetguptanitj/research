L4 · Compute Query Engine Federation Iceberg 2026-05-22

Trino: Distributed SQL Query Engine

A disaggregated MPP SQL engine that federates queries across disparate data sources — object storage, relational databases, key-value stores — without moving data, exposing every source as a typed catalog through a plugin-based connector SPI.

The short version: Trino (born at Facebook as Presto, 2012) runs a single coordinator that plans queries and distributes work to N workers via HTTP REST. Every data source is a plugin implementing the Connector SPI. It is the canonical query engine for open lakehouses: Iceberg connector + IRC catalog + S3 = no proprietary lock-in at any layer. Not a replacement for OLTP or streaming.

L3 · Catalog Open Standard Credential Vending Auth 2026-05-21

The Iceberg REST Catalog (IRC)

An open HTTP/JSON API specification that decouples query engines from catalog implementations — and, in doing so, makes the catalog the natural home for authentication, authorization, and short-lived storage credential vending in the open lakehouse.

The short version: IRC is a protocol, not a product — the OpenAPI contract in apache/iceberg that any engine can speak and any catalog can implement. Its credential-vending endpoint is what lets Trino, Spark, and others read S3 without holding long-lived keys. Implementations include Polaris, Unity Catalog, Nessie, Gravitino, Lakekeeper, and AWS Glue.
