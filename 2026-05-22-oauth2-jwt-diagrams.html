3. JWT Appears Twice — The Common Confusion

JWT is used as the input credential AND often as the output access token format — two different JWTs.

4. Mental Model — Separation of Concerns

OAuth2 is the protocol, JWT is the format. They operate at different layers.

At registration time the client hands over its public key via static upload or a JWKS URI it hosts.

6. JWKS Endpoint — What's Inside

The JWKS endpoint returns a JSON array of public keys. The kid field links each key to JWTs that reference it in their header.

7. Zero-Downtime Key Rotation via JWKS URI

When the auth server sees an unknown kid, it re-fetches the JWKS endpoint to pick up new keys — no re-registration needed.

sequenceDiagram participant C as Client participant AS as Auth Server participant JWKS as Client's JWKS Endpoint Note over C: Rotates to new key pair.
Updates JWKS endpoint
with new kid: key-id-2.
Old kid: key-id-1 still present. C->>AS: POST /token
assertion JWT { kid: key-id-2 } AS->>AS: Look up kid: key-id-2
in cache — NOT FOUND AS->>JWKS: GET /.well-known/jwks.json JWKS-->>AS: { keys: [ key-id-1, key-id-2 ] } AS->>AS: Cache new keys AS->>AS: Verify JWT signature\nusing key-id-2 AS-->>C: 200 OK — access token issued Note over AS,JWKS: Old key-id-1 stays in JWKS
until in-flight tokens signed
with it have expired.
